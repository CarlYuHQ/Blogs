一、值迭代算法 — Section 4.1

1.1 算法描述

值迭代算法就是上一章压缩映射定理（Theorem 3.3）所建议的求解贝尔曼最优方程的迭代算法：

$$v_{k+1} = \max_{\pi \in \Pi} (r_\pi + \gamma P_\pi v_k), \quad k = 0, 1, 2, \ldots$$

根据压缩映射定理保证，当 k→∞k \to \inftyk→∞ 时，vkv_kvk​ 收敛到最优状态值，πk\pi_kπk​ 收敛到最优策略。

每次迭代包含两个步骤：

第一步：策略更新（Policy Update）。找到使下式最大化的策略：

$$\pi_{k+1} = \arg\max_\pi (r_\pi + \gamma P_\pi v_k)$$

第二步：值更新（Value Update）。计算新的值：

$$v_{k+1} = r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v_k$$

1.2 逐元素形式与实现

上面的描述是矩阵-向量形式。为了实现该算法，需要研究其逐元素（elementwise）形式。

考虑时间步 kkk 和状态 sss：

策略更新步骤的逐元素形式为：

$$\pi_{k+1}(s) = \arg\max_\pi \sum_a \pi(a|s) \underbrace{\left[\sum_r p(r|s,a) r + \gamma \sum_{s’} p(s’|s,a) v_k(s’)\right]}_{q_k(s,a)}, \quad s \in \mathcal{S}$$

我们在第三章 Section 3.3.1 已经证明，最优解为：

$$\pi_{k+1}(a|s) = \begin{cases} 1, & a = a_k^*(s) \ 0, & a \neq a_k^*(s) \end{cases}$$

其中 ak∗(s)=arg⁡max⁡aqk(s,a)a_k^*(s) = \arg\max_a q_k(s, a)ak∗​(s)=argmaxa​qk​(s,a)。如果 arg⁡max⁡aqk(s,a)\arg\max_a q_k(s, a)argmaxa​qk​(s,a) 有多个解，可以任选一个，不影响算法收敛。由于新策略选择 qk(s,a)q_k(s, a)qk​(s,a) 最大的动作，因此称为贪心策略（greedy policy）。

值更新步骤的逐元素形式为：

$$v_{k+1}(s) = \sum_a \pi_{k+1}(a|s) \underbrace{\left[\sum_r p(r|s,a) r + \gamma \sum_{s’} p(s’|s,a) v_k(s’)\right]}_{q_k(s,a)}, \quad s \in \mathcal{S}$$

将贪心策略代入，得到：

$$v_{k+1}(s) = \max_a q_k(s, a)$$

整个过程可以概括为：

$$v_k(s) \to q_k(s, a) \to \text{新贪心策略 } \pi_{k+1}(s) \to \text{新值 } v_{k+1}(s) = \max_a q_k(s, a)$$

注意：vkv_kvk​ 不是状态值。虽然 vkv_kvk​ 最终收敛到最优状态值，但它一般不满足任何策略的贝尔曼方程。例如，它不满足 vk=rπk+1+γPπk+1vkv_k = r_{\pi_{k+1}} + \gamma P_{\pi_{k+1}} v_kvk​=rπk+1​​+γPπk+1​​vk​ 或 vk=rπk+γPπkvkv_k = r_{\pi_k} + \gamma P_{\pi_k} v_kvk​=rπk​​+γPπk​​vk​。同理，qkq_kqk​ 也不是动作值。

二、策略迭代算法 — Section 4.2

2.1 算法分析

策略迭代不同于值迭代，它不是直接求解贝尔曼最优方程，但其思想在强化学习中极为重要。

策略迭代也是迭代算法，每次迭代包含两个步骤：

第一步：策略评估（Policy Evaluation, PE）。计算给定策略的状态值，即求解贝尔曼方程：

$$v_{\pi_k} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k}$$

其中 πk\pi_kπk​ 是上一次迭代得到的策略，vπkv_{\pi_k}vπk​​ 是待求的状态值。

第二步：策略改进（Policy Improvement, PI）。根据计算得到的状态值改进策略：

$$\pi_{k+1} = \arg\max_\pi (r_\pi + \gamma P_\pi v_{\pi_k})$$

关于这个算法，有三个自然的问题：

  1. 在策略评估步骤中，如何求解 vπkv_{\pi_k}vπk​​？
  2. 在策略改进步骤中，为什么新策略 πk+1\pi_{k+1}πk+1​ 比 πk\pi_kπk​ 更好？
  3. 为什么算法最终能收敛到最优策略？

问题 1：如何求解 vπkv_{\pi_k}vπk​​？

我们在第二章介绍了两种方法：

方法一：封闭解

$$v_{\pi_k} = (I - \gamma P_{\pi_k})^{-1} r_{\pi_k}$$

适合理论分析，但需要计算矩阵逆，实现效率低。

方法二：迭代算法

$$v_{\pi_k}^{(j+1)} = r_{\pi_k} + \gamma P_{\pi_k} v_{\pi_k}^{(j)}, \quad j = 0, 1, 2, \ldots$$

从任意初始猜测 vπk(0)v_{\pi_k}^{(0)}vπk​(0)​ 出发，vπk(j)→vπkv_{\pi_k}^{(j)} \to v_{\pi_k}vπk​(j)​→vπk​​ 当 j→∞j \to \inftyj→∞。

有趣的是，策略迭代是一个迭代算法，而其策略评估步骤中又嵌套了另一个迭代算法。理论上，内部迭代需要无限步才能收敛到真实状态值，但实际中不可能实现。通常在满足某个准则时终止，例如 ∥vπk(j+1)−vπk(j)∥\|v_{\pi_k}^{(j+1)} - v_{\pi_k}^{(j)}\|∥vπk​(j+1)​−vπk​(j)​∥ 小于阈值或 jjj 超过预设值。即使只能得到近似值，也不会影响算法收敛（原因在截断策略迭代部分说明）。

问题 3：为什么策略迭代能找到最优策略？

策略迭代生成两个序列：策略序列 {π0,π1,…}\{\pi_0, \pi_1, \ldots\}{π0​,π1​,…} 和状态值序列 {vπ0,vπ1,…}\{v_{\pi_0}, v_{\pi_1}, \ldots\}{vπ0​​,vπ1​​,…}。

根据引理 4.1，策略不断改进：

$$v_{\pi_0} \leq v_{\pi_1} \leq v_{\pi_2} \leq \cdots \leq v_{\pi_k} \leq \cdots \leq v^*$$

由于 vπkv_{\pi_k}vπk​​ 单调递增且有上界 v∗v^*v∗，由单调收敛定理知 vπkv_{\pi_k}vπk​​ 收敛到某个值 v∞v_\inftyv∞​。

定理 4.1（策略迭代的收敛性）：策略迭代算法生成的状态值序列收敛到最优状态值 v∗v^*v∗，策略序列收敛到最优策略。

3.3 三种算法的关系

Figure 4.5 直观展示了三种算法的收敛速度关系：

  • 值迭代收敛最慢（策略评估仅 1 步）
  • 策略迭代收敛最快（策略评估运行至收敛）
  • 截断策略迭代介于两者之间

截断策略迭代的优点：

  • 相比策略迭代：策略评估仅需有限次迭代，计算效率更高
  • 相比值迭代：多运行几次策略评估迭代可以加速收敛

四、总结 — Section 4.4

本章介绍了三个寻找最优策略的算法：

算法 每次迭代的两个步骤 特点
值迭代 值更新 + 策略更新 压缩映射定理的直接实现
策略迭代 策略评估 + 策略改进 嵌套迭代，收敛更快
截断策略迭代 截断策略评估 + 策略改进 前两者的统一框架

三个算法的共同特点：每次迭代都包含值更新和策略更新两个步骤。值与策略交替更新的思想在强化学习中广泛存在，称为广义策略迭代（Generalized Policy Iteration）。

本章算法需要已知系统模型（动态规划算法）。从第五章开始，将学习无模型（model-free）强化学习算法——它们可以看作本章算法的推广。

五、问答 — Section 4.5

Q：值迭代算法是否保证找到最优策略？
A：是的。值迭代就是压缩映射定理求解贝尔曼最优方程的算法，收敛性由压缩映射定理保证。

Q：值迭代中的中间值是状态值吗？
A：不是。这些值不保证满足任何策略的贝尔曼方程。

Q：策略迭代算法包含哪些步骤？
A：每次迭代包含策略评估（求解贝尔曼方程得到当前策略的状态值）和策略改进（更新策略使新策略有更大的状态值）。

Q：策略迭代中是否嵌套了另一个迭代算法？
A：是的。在策略评估步骤中，需要迭代求解当前策略的贝尔曼方程。

Q：策略迭代中的中间值是状态值吗？
A：是的。因为它们是当前策略贝尔曼方程的解。

Q：策略迭代是否保证找到最优策略？
A：是的。本章给出了严格的收敛性证明。

Q：截断策略迭代与策略迭代的关系？
A：截断策略迭代从策略迭代得到，只是在策略评估步骤中执行有限次迭代。

Q：截断策略迭代与值迭代的关系？
A：值迭代是截断策略迭代的极端情况，策略评估步骤仅运行 1 次迭代。

Q：截断策略迭代中的中间值是状态值吗？
A：不是。只有策略评估步骤运行无限次才能得到真实状态值，有限次迭代只能得到近似值。

Q：策略评估步骤应该运行多少次迭代？
A：一般准则：运行少量但不太多的迭代。少量迭代可以加速整体收敛，但太多迭代不会显著加速收敛。

Q：什么是广义策略迭代？
A：广义策略迭代不是一个具体算法，而是值与策略交替更新的通用思想。大多数强化学习算法都落入广义策略迭代的范畴。

Q：什么是有模型和无模型强化学习？
A：本章算法称为动态规划算法（需要系统模型）。强化学习分为：

  • 有模型（model-based）：使用数据估计系统模型，在学习过程中使用该模型
  • 无模型（model-free）：在学习过程中不涉及模型估计
