主页与博客联动重构纪要

2026-05-27- 2026-05-28
1.6k- 6m
随笔-分享博客主页重构

这篇我想用“复盘 + 操作手册”的方式，完整记录这段时间我对个人主页和随笔博客做的联动重构。

说白了这次重构只有三个目标：

  1. 写作只在 Hexo 做一次，不用重复搬运到主页；
  2. 主页最新动态尽量自动更新，少手工；
  3. Routine 能真正日更并看得到统计，不是摆设。

这篇会写得很细，尽量做到：以后我自己看也能快速恢复现场。

一、先讲结论：现在这套架构到底怎么跑

我现在维护的是两个仓库：

  • Hexo 博客仓库（内容生产）：D:\Hexo-Blog\blog
  • Jekyll 主页仓库（内容分发 + 个人主页）：D:\pkuhub\CarlYuHQ.github.io

线上地址对应如下：

  • 主页：https://carlyuhq.github.io/
  • 博客：https://carlyuhq.github.io/Blogs/
  • Feed：https://carlyuhq.github.io/Blogs/atom.xml

这意味着什么？

  • 博客正文只在 Hexo 维护
  • 主页只做“展示聚合”，不再维护第二套文章
  • 顶栏 Blogs 永远外链到 /Blogs/
  • 主页“最新动态”通过脚本合并：
    • Hexo atom feed（自动）
    • site_updates.yml（手动写本站更新）

二、这次改了什么（详细）

1）主页“最新动态”模块

我把首页动态改成“时间轴卡片”结构，并统一了每条数据的字段：

  • 日期
  • 标题/标签
  • 摘要
  • 来源（本站更新 / 来自随笔博客）
  • 跳转按钮

核心文件（主页仓库）：

  • _includes/ac-updates.html（展示）
  • _data/site_updates.yml（手动更新源）
  • _data/latest_updates.json（最终渲染源）
  • scripts/sync-hexo-feed.js（合并脚本）

我还修了一个很容易忽略的问题：
以前摘要多行内容会被截断，现在解析规则已经修复，多段摘要不会莫名只显示第一句。

2）Routine 模块（这是我最想要的）

首页 Routine 现在是可操作、可统计、可日更的：

  • 任务项：LeetCode / 阅读 / 运动
  • 支持命令行更新 done/undo
  • LeetCode streak 自动维护
  • 增加统计面板：
    • 近30天完成率
    • 近30天打卡天数
    • 本月完成率
    • 近30天柱状图
  • 统计面板展开/收起状态本地记忆（localStorage）
  • 日期改成前端按“访问当天”渲染，避免卡在构建日

核心文件：

  • _includes/ac-routine.html
  • _data/routine.yml
  • scripts/routine.js
  • assets/js/ac-interactions.js

3）Notes / Links / 导航联动

  • 补充“观点与本站发展”区块内容
  • 增加锚点，支持首页动态直接跳转到指定段落
  • 修正 Links 友链地址，统一到当前博客域名

4）样式与显示稳定性（这块最折腾）

这一块踩坑很多，核心矛盾是：

  • 首页和子页布局结构不同（single vs archive）
  • 横幅图片本身比例非常扁
  • 浏览器缩放时会出现子像素渲染异常
  • 某些 class 名可能被插件策略误伤

处理过的问题包括：

  • 页眉/页脚横幅在部分缩放比“消失/只剩线”
  • 头像偶发不显示
  • 统计按钮点击异常
  • 代码块显示发灰看不清

最终做法：

  • 统一插入与容器规则
  • 增加前端兜底逻辑（异常高度自动恢复）
  • 按钮绑定顺序前置 + 防重复绑定
  • 代码块样式单独覆盖，避免被全局 table 样式误伤

五、自动同步到底多久（我这边的体感）

这是最关心的点，我给一个更实际的范围：

1）博客发布（Hexo）

  • generate：3~10 秒
  • deploy 推送：10~60 秒
  • 博客站点可访问：30 秒 ~ 3 分钟

2）主页动态同步

现在主页 sync-feed.yml 已经改成每小时跑一次。
所以在 dispatch 没触发时，最坏就是一小时内更新。

如果 dispatch 触发成功，一般 2~6 分钟可见。
如果只靠每小时定时，则取决于你发文时点与下一个整点的间隔。

3）卡住时怎么查（按顺序）

  1. 博客文章链接能不能打开
  2. Feed 有没有这篇：/Blogs/atom.xml
  3. 主页仓库 Actions 是否成功
  4. 首页是否更新（强刷 Ctrl + F5）

六、这次重构后，我自己的感受

这次改完，最大的变化不是“页面更花”，而是维护心智负担下降了：

  • 写博客只管 Hexo，不再双端维护
  • 主页动态有自动链路，不至于忘更新
  • Routine 终于不是摆件，能看趋势
  • 出问题有兜底命令，不会慌

后面我还想继续做两件事：

  • 加一个 routine report 30 命令，直接输出月报
  • 给主页加“最近一次同步时间”提示，减少猜测

总之，折腾是挺折腾，但现在这套已经能稳定跑起来了。
