Agentic-RL 详解：从 SFT 到 GRPO 的智能体训练

2026-05-29- 2026-06-17
3.1k- 11m
课程笔记-学习AI Agent强化学习机器学习分享

本文基于 DataWhale Hello-Agents「Agentic-RL」章节整理与扩展，并结合我此前强化学习系列笔记，尝试把「LLM 后训练」和「Agent 多步决策」放在同一张图里理解。

当我们已经能搭建 ReAct Agent、接上 MCP 工具、做多 Agent 协作之后，会遇到更棘手的问题：

  • 模型会不会「看起来会推理」，但一换题型就崩？
  • 工具调用成功率不稳定，靠 prompt 很难系统性提升？
  • 多步任务里，中间步骤错了，最终答案还能不能救回来？

这就是 Agentic-RL（基于强化学习的智能体训练） 要解决的问题：
把 LLM 当作可学习策略，放进 Agent 的感知-决策-执行循环，用 RL 优化长期任务表现。

一、从 LLM 训练到 Agentic-RL：问题是怎么变难的？

1.1 传统 SFT 的三类局限

监督微调（SFT）很强，但对 Agent 任务有三个天然短板：

  1. 数据上限：模型主要在「模仿」标注数据，难超越标注质量
  2. 缺乏探索：不会主动尝试不同路径，只会复现人类示范
  3. 难优化长期目标：中间步骤是否正确，SFT 很难精细约束

1.2 一个数学推理例子（GSM8K 风格）

可映射到 RL 框架：

RL 概念 在 LLM Agent 中的对应
智能体 基于 LLM 的推理系统
环境 数学题 + 自动判题器
状态 题目描述 + 已生成推理步骤
动作 生成下一步推理 / 最终答案
奖励 答案是否正确（如正确 +1，错误 0）

RL 的关键在于：模型可以自己探索多条推理路径，根据结果反馈学习「哪条路更好」，而不只是背标准答案。

这就是 Agentic-RL 的核心思想。

二、LLM 训练全景：预训练 → 后训练 → Agentic-RL

2.1 预训练（Pretraining）

目标：学习语言规律与世界知识。

  • 数据：海量无标注文本（TB 级）
  • 任务：Next Token Prediction / Causal LM
  • 结果：模型会「续写」，但不一定「听话」

预训练模型像一本读过很多书的「语言天才」，但还不懂：

  • 如何遵循指令
  • 如何拒绝不当请求
  • 如何以对话助手的方式交互

2.2 后训练（Post-training）

目标：对齐人类偏好，让模型从「会预测」变成「会帮忙」。

常见路线：

  1. SFT：用高质量指令数据教格式与基本能力
  2. RLHF：用人类偏好训练奖励模型，再 RL 优化
  3. RLAIF：用强模型（如 GPT-4）替代人类标注，降本增效

实验上，RLAIF 效果可接近甚至超过 RLHF，但成本显著更低。

2.3 从 PBRFT 到 Agentic-RL

传统偏好强化微调（PBRFT）更关注单轮回答质量：

  • 输入：一个问题
  • 输出：一个完整回答
  • 奖励：回答好不好

Agentic-RL 关注多步任务完成质量：

  • 输入：一个复杂目标
  • 过程：多步行动（推理、调工具、读写环境）
  • 奖励：长期累积回报

对比例子：解释 RL vs 分析 GitHub 仓库

PBRFT 场景

  • 用户：请解释什么是强化学习
  • 模型：一次性生成长回答
  • 奖励：回答质量评分

Agentic-RL 场景

  • 用户：分析某 GitHub 仓库代码质量
  • Agent 步骤：
    1. 调 GitHub API 获取仓库结构（+0.1）
    2. 读取核心代码文件（+0.1）
    3. 做质量分析（+0.2）
    4. 输出高质量报告（+0.6）
  • 总奖励：1.0（过程 + 结果）

关键差异：

维度 PBRFT Agentic-RL
任务类型 单轮问答 多步序贯决策
状态 静态 prompt 动态演化状态
动作 文本生成 文本 + 工具 + 环境操作
优化目标 单步回答质量 长期任务成功

三、Agentic-RL 要提升的六大核心能力

Hello-Agents 把 Agentic-RL 的目标总结为六项能力：

  1. 推理（Reasoning）：多步逻辑推导，学习推理策略
  2. 工具使用（Tool Use）：学会何时调用、如何调用外部工具
  3. 记忆（Memory）：决定记什么、忘什么、何时检索
  4. 规划（Planning）：为长期目标制定行动序列，允许「绕路」收集信息
  5. 自我改进（Self-Improvement）：复盘错误、修正策略
  6. 感知（Perception）：理解多模态信息，结合视觉/文本做决策

其中「规划」和「自我改进」尤其重要：

  • CoT 往往是线性思考，难回溯
  • Prompt 模板是静态规划，难适应新环境
  • RL 可以通过试错学习「先做什么、后做什么」

四、数据与奖励：Agentic-RL 的两块地基

4.1 为什么选 GSM8K 作为入门任务？

GSM8K（Grade School Math 8K） 很适合做 Agentic-RL 入门：

  • 答案可自动判定（适合程序化奖励）
  • 需要 2~8 步推理（典型多步任务）
  • 推理能力可迁移到其他领域

数据规模（常见版本）：

  • 训练集：约 7,473
  • 测试集：约 1,319
  • 难度：小学应用题（2~8 年级）

4.2 三种数据格式：原始 / SFT / RL

原始格式

  • question
  • answer（含中间步骤）

SFT 格式（教格式 + 教过程）

作用：让模型学会

  • 对话模板
  • 分步推理写法
  • 最终答案格式

RL 格式（只给问题 + 真值）

关键：不提供标准推理过程，逼模型自己探索推理路径。

4.3 奖励函数设计：决定模型学到什么

常见三类奖励

1）准确率奖励（Accuracy Reward）

  • 答案完全正确 → 高分
  • 优点：简单直接
  • 缺点：奖励稀疏，训练初期信号弱

2）长度惩罚（Length Penalty）

  • 鼓励简洁，抑制废话
  • 优点：控制 token 成本
  • 缺点：可能压制必要推理步骤

3）步骤奖励（Step Reward）

  • 奖励「Step 1 / Step 2 / Final Answer」等结构化推理
  • 优点：更可解释、易调试
  • 缺点：可能诱导「为了凑步骤而凑步骤」

好奖励函数 vs 坏奖励函数

好奖励函数 坏奖励函数
成功标准清晰 只在最后给奖励，中间无反馈
梯度信号稳定 容易被「作弊」策略钻空子
方差可控 多目标互相冲突
易于调参组合 方差过大导致训练不收敛

五、SFT：Agentic-RL 的「起跑线」

5.1 为什么几乎必须先 SFT？

直接 RL 往往会失败，因为模型连基本输出格式都没有：

  • 不会写 Step 1 / Final Answer
  • 不会按任务模板回答
  • 探索空间过大，奖励几乎全为 0

SFT 的作用：

  1. 学会输出格式
  2. 学会基础推理模板
  3. 建立合理初始策略
  4. 缩小 RL 探索空间

可以把它看成：SFT 教会模型「怎么开始解题」，RL 教会模型「怎么解得更好」。

5.2 LoRA：消费级 GPU 也能微调

全量微调成本高。以 Qwen3-0.6B 为例：

  • FP16 全量微调约需 12GB 显存
  • 更大模型（7B/13B）在消费级 GPU 上几乎不可行

LoRA（Low-Rank Adaptation） 的核心思想：

微调时的参数更新可以用低秩矩阵近似，只训练少量增量参数，冻结原模型权重。

优势：

  • 显存占用显著下降
  • 训练更快
  • 部署时可合并或外挂 LoRA 权重
  • 一定程度缓解过拟合

代价：效果通常略弱于全量微调，但性价比极高。

5.3 SFT 阶段重点看什么指标？

  • Accuracy：最终答案正确率
  • Average Reward：综合奖励（若已接入奖励函数）
  • Reasoning Quality：推理是否清晰（可人工抽检）

六、GRPO：比 PPO 更适合 LLM 的 RL 算法

SFT 之后，模型会「像样地解题」，但常常只是模仿训练集。
要突破数据上限，需要 RL。

6.1 为什么 PPO 在 LLM 上很重？

PPO（Proximal Policy Optimization）经典但重：

  • 需要 Value Model（价值网络）
  • 通常要维护 4 个模型：Policy / Reference / Value / Reward
  • 工程复杂、显存压力大
  • 容易出现奖励崩塌、策略退化

6.2 GRPO 的核心简化

GRPO（Group Relative Policy Optimization） 是面向 LLM 的 PPO 变体，关键变化：

  1. 去掉 Value Model
  2. 用组内相对奖励替代绝对奖励
  3. 只保留 Policy Model + Reference Model
  4. 训练更稳定，显存更友好

直觉理解：

  • PPO：这个回答绝对好不好？
  • GRPO：这组候选里，谁相对更好？

相对奖励能降低方差，让模型学习「比组内平均水平更好」的输出，而不是被稀疏绝对奖励困住。

对 LLM 训练，GRPO 往往是更实用的默认选择。

6.3 GRPO 训练过程（概念版）

  1. 对同一 prompt 采样一组候选回答
  2. 用奖励函数给每个候选打分
  3. 做组内归一化，得到相对优势
  4. 用 clipped objective 更新策略，防止更新过猛
  5. Reference Model 提供 KL 约束，避免偏离 SFT 太远

七、评估与迭代：别只看一个准确率

训练完不能只看 Accuracy。Agentic-RL 需要系统化评估：

7.1 四类常见错误

  1. 计算错误：思路对，算术错（如 48/2=25）
  2. 推理错误：步骤顺序或逻辑错（先加后除）
  3. 理解错误：没抓住题意（问「总共」却只算一部分）
  4. 格式错误：答案对但不符合输出格式

7.2 持续迭代闭环

这是 Agentic-RL 真正「能上线」的关键，而不是一次性跑通脚本。

7.3 端到端训练 Pipeline

典型流程：

  1. 数据准备（原始 → SFT / RL 格式）
  2. SFT 训练（LoRA）
  3. SFT 评估
  4. GRPO 训练
  5. GRPO 评估
  6. 部署与在线监控

7.4 超参数调优方法

  • Grid Search：遍历组合，简单但贵
  • Random Search：随机采样，效率更高
  • Bayesian Optimization：用代理模型指导搜索，更智能

八、Agentic-RL 与传统 RL 课程如何对齐？

我前面写过强化学习系列（MDP、贝尔曼方程、MC、TD、策略梯度等）。对应关系可以这样理解：

传统 RL 概念 Agentic-RL 中的实例
状态 (s) 题目 + 历史推理 + 工具返回
动作 (a) 下一个 token / tool call / 环境操作
策略 (\pi(a s))
奖励 (r) 正确性、步骤质量、长度惩罚等
回报 (G) 多步任务累积奖励
策略梯度 GRPO / PPO 类目标函数

所以 Agentic-RL 不是「另一套玄学」，而是把 LLM 放进 RL 框架后的工程化实践。

九、学习路线建议（结合 Hello-Agents）

基础阶段

  1. 复习 RL 基础（MDP、策略梯度、PPO 思想）
  2. 理解 LLM 训练链路（预训练 / SFT / RLHF / RLAIF）
  3. 跑通 Hello-Agents 示例（GSM8K + SFT + GRPO）

进阶阶段

  1. 读 TRL 实现，理解训练细节
  2. 自定义数据集（不止数学题）
  3. 自定义奖励（工具调用成功率、步骤合法性等）
  4. 系统调参（学习率、KL 系数、组大小、奖励权重）

高级阶段

  1. 长链路推理任务
  2. 工具学习（Tool-use RL）
  3. 多智能体协作 + RL
  4. 跟进 DeepSeek-R1、OpenAI o 系列等推理模型训练思路

十、小结

Agentic-RL 的本质，是把 LLM 从「会聊天的模型」推向「能完成任务的智能体策略」：

  • SFT 解决「会不会做题」
  • RL（GRPO） 解决「能不能做得更好」
  • 奖励设计 决定「到底学成什么样」
  • 评估闭环 决定「能不能持续变强」

如果只记一句话：

Agentic-RL = Agent 架构 + 可验证任务 + 合理奖励 +（SFT 打底 + GRPO 优化）
