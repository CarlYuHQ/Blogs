Docker 介绍

Docker 简介

Docker 是一个开源的应用容器引擎，让开发者可以将应用程序和所有依赖打包到一个轻量级、可移植的容器中，然后在任何支持 Docker 的环境中运行。

在传统的项目开发中，开发者经常遇到环境不一致的问题：代码在本地开发环境运行正常，但在测试或生产环境却出现各种错误，原因可能是操作系统版本、依赖库版本或配置差异。Docker 的核心目标就是解决这些问题，通过容器化技术将应用及其运行环境打包在一起，确保应用在不同环境中表现一致。

Docker 有几个显著特点：

  • 轻量性：容器共享宿主机的操作系统内核，比传统虚拟机更小且启动更快
  • 可移植性：容器可以在任何支持 Docker 的平台上运行，彻底解决”在我机器上能跑”的问题
  • 隔离性：每个容器拥有独立的文件系统、网络和进程空间，应用之间互不干扰
  • 标准化：提供统一的接口和工具链，构建、分发和运行容器变得简单高效

Docker 采用了客户端-服务器架构。Docker 客户端负责发送命令，Docker 守护进程（服务端）负责执行。它们通过 UNIX 套接字或网络接口通信，使用 REST API 交换数据。

Docker 核心概念

  • 容器（Container）：轻量级虚拟化技术，共享操作系统内核但保持独立运行环境。容器之间相互独立，各自拥有文件系统、网络和进程空间。
  • 镜像（Image）：创建容器的模板，包含运行应用所需的代码、库和配置文件。镜像采用分层存储结构，内容在构建后不再变动。
  • Dockerfile：定义镜像构建过程的文本文件，由一系列指令组成（FROM 指定基础镜像、RUN 执行命令、COPY 复制文件等），使镜像创建可重复且透明。
  • 镜像仓库（Registry）：集中存储和分发镜像的地方。最常用的公共仓库是 Docker Hub，也可以搭建私有仓库。

工作流程：编写 Dockerfile → 构建镜像 → 推送到仓库 → 在任何机器上拉取并运行容器。

Docker 和虚拟机

Docker 容器直接运行在宿主机内核上，不需要模拟完整操作系统，启动更快、占用资源更少。虚拟机通过 Hypervisor 模拟完整硬件环境，每个虚拟机需要独立操作系统，启动慢且占用大量内存。虚拟机隔离性更强，但 Docker 的隔离性对大多数场景已足够。

Dockerfile

Dockerfile 简介

Dockerfile 是构建 Docker 镜像的文本文件，包含一系列指令，每条指令在镜像中新建一层。使用 Dockerfile 可以自动构建镜像，在不同机器上得到完全一致的结果。

Dockerfile 常用指令

指令 描述
FROM 指定基础镜像（必须是第一条指令）
LABEL 添加镜像元数据（键值对）
RUN 构建时执行的命令（每条创建一个新层）
CMD 容器启动时默认命令（可被覆盖）
ENTRYPOINT 容器启动主命令（不可被覆盖）
EXPOSE 声明容器监听端口
ENV 设置环境变量
COPY 将文件或目录复制到镜像中
ADD 类似 COPY，支持远程 URL 和自动解压
WORKDIR 指定工作目录
VOLUME 创建挂载点
USER 切换执行用户
ARG 定义构建时变量
HEALTHCHECK 定义容器健康检查命令

FROM 指令：指定基础镜像，推荐使用官方最小化镜像（如 alpine）。支持多阶段构建（FROM ... AS builder）。

RUN 指令：两种格式，Shell 格式（RUN <command>）和 Exec 格式（RUN ["executable", "param"]）。建议用 && 合并多个命令减少层数：

CMD vs ENTRYPOINT：CMD 可被 docker run 后的命令覆盖，ENTRYPOINT 不会。推荐组合使用：ENTRYPOINT 指定程序，CMD 提供默认参数。

COPY vs ADD：COPY 只复制本地文件，ADD 额外支持远程 URL 下载和 tar 自动解压。推荐优先使用 COPY。
